📜 1. JS Basics & Variables

// let: Block-scoped, re-assignable
let count = 0;
count = 1;

// const: Block-scoped, read-only
const user = { name: "Alice" };
user.name = "Bob"; // Allowed! (Mutating object)

// Primitive Types
const str = "Hello";    // String
const num = 42;         // Number
const isTrue = true;    // Boolean
const arr = [1, 2, 3];  // Array

💡 Tip: Use const by default, and use let only if you know the value will be reassigned.

🖥️ 2. DOM Manipulation

// Selecting elements
const btn = document.querySelector('#myBtn');
const items = document.querySelectorAll('.item');

// Event Listeners
btn.addEventListener('click', (e) => {
    e.preventDefault();
    console.log('Button clicked!');
});

// Creating and Appending elements
const div = document.createElement('div');
div.textContent = 'New Element';
div.classList.add('highlight');
document.body.appendChild(div);

💡 Tip: querySelector selects the first match, whereas querySelectorAll returns a NodeList of all matches.

✨ 3. ES6+ Features

// Destructuring
const { name, age } = user;
const [first, second] = arr;

// Spread & Rest Operator
const newArr = [...arr, 4, 5];
const newUser = { ...user, age: 30 };

// Arrow Functions
const add = (a, b) => a + b;

// Template Literals
const greeting = `Hello, ${name}!`;

💡 Tip: Destructuring and the spread operator make state manipulation much cleaner, especially in modern frameworks.

⏳ 4. Asynchronous JS

// Promises
const wait = (ms) => new Promise(resolve => 
    setTimeout(resolve, ms)
);

// Async / Await
async function executeTask() {
    try {
        console.log('Task started...');
        await wait(2000); // Waits for 2 seconds
        console.log('Task finished!');
    } catch (err) {
        console.error('Error:', err);
    }
}

💡 Tip: async/await is syntactic sugar over Promises, allowing async code to look and behave synchronously.

🌐 5. API Requests (Fetch)

async function fetchUser() {
    try {
        const res = await fetch('https://api.example.com/user');
        
        // fetch doesn't throw on HTTP errors
        if (!res.ok) {
            throw new Error(`HTTP Error: ${res.status}`);
        }
        
        const data = await res.json();
        console.log('User data:', data);
    } catch (error) {
        console.error('Fetch failed:', error);
    }
}

💡 Tip: Always check res.ok because fetch only rejects on network failures, not on 404 or 500 responses.
