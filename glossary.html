התמונה הגדולה: איך הכל מתחבר? 🧩

הקוד (Code): המוח וההיגיון של האפליקציה. כאן אתה כותב את החוקים – מה קורה, מתי זה קורה, ומה התנאים.
השרת (Server / FastAPI): המתווך החכם ("המלצר"). מקבל בקשות מהלקוח, בודק שהן חוקיות, לוקח אותן אל "המטבח" (הקוד שלך) כדי לעבד אותן, ואז מחזיר ללקוח את התשובה בצורה מסודרת (לרוב כ-JSON).
הראוטר (Router): מרכזיית הניתוב ("פקיד קבלה"). מסתכל על כתובת הבקשה (כמו /items) ועל סוג הבקשה (GET/POST), ומנתב אותה לפונקציה הספציפית בקוד שלך שאמורה לטפל בה.
קורסור (Cursor): האצבע שעוקבת אחרי השורות בדאטה-בייס. כשמסד הנתונים מכיל מיליוני רשומות, ה-Cursor מאפשר לפייתון לקרוא אותן שורה אחרי שורה כדי שהשרת לא יקרוס מעומס זיכרון.
מסד הנתונים (Database): מחסן הזיכרון לטווח ארוך של המערכת. שם שומרים טבלאות SQL כדי שהמידע לא יימחק כשמכבים את השרת.
לוגר (Logger): הקופסה השחורה של השרת. מתעד כל בקשה שנכנסת וכל שגיאה שקרתה. קריטי למציאת תקלות (Debugging).

מילון מושגים מהיר (עם דוגמאות) 📝

שאילתה (Query)
בקשה למידע ממסד הנתונים. זוהי פקודת SQL טהורה שנשלחת למסד הנתונים כדי לשלוף או לשנות נתונים.
query = "SELECT * FROM users WHERE age > %s"
cursor.execute(query, (18,))
פרמטר נתיב (Path Parameter)
משתנה שהוא חלק בלתי נפרד מכתובת ה-URL, נועד לזיהוי משאב ספציפי וחובה לספק אותו.
@app.get("/items/{item_id}") # item_id is the path parameter
def read_item(item_id: int):
    return {"item_id": item_id}
פרמטר שאילתה (Query Parameter)
מופיע בסוף ה-URL אחרי סימן שאלה (?), לרוב משמש לסינון וחיפוש (אינו חלק מהנתיב עצמו).
# URL: /items/?skip=0&limit=10
@app.get("/items/")
def read_items(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}
שמירת שינויים במסד נתונים (Commit)
פקודה שמודיעה למסד הנתונים שהשינויים שעשינו (הוספה/עדכון/מחיקה) סופיים ויש לשמור אותם הרדיסק. בלעדיה השינויים ייעלמו.
cursor.execute("INSERT INTO users (name) VALUES ('Natan')")
connection.commit() # חובה!
מודל אימות (Pydantic BaseModel)
מחלקה שמוודאת שהנתונים הנכנסים ויוצאים תקינים לחלוטין לפני שהם נוגעים בלוגיקה שלנו.
from pydantic import BaseModel

class User(BaseModel):
    name: str
    age: int
גוף הבקשה (Request Body)
הנתונים שהלקוח שולח (לרוב כ-JSON) ביצירת אובייקט חדש (POST) או עדכונו (PUT). הראוטר אוסף אותם מה-Body.
@app.post("/users/")
def create_user(user: User): # ה'user' מגיע מגוף הבקשה (Body)
    return user
תבנית רפוזיטורי (Repository)
מחלקה שמרכזת את כל פקודות ה-SQL. הראוטר פונה רק אליה ולא כותב SQL בעצמו.
class UserRepository:
    def get_user_by_id(self, user_id):
        cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,))
        return cursor.fetchone()
ביצוע (Execute)
פעולת שיגור השאילתה אל המסד דרך ה-Cursor.
cursor.execute("DELETE FROM users WHERE id = %s", (5,))
חריגות ושגיאות (Exceptions)
שליטה על התרסקויות קוד. אם משהו קורס ב-try, בלוק ה-except יתפוס את זה כדי שהתוכנה לא תעצור לגמרי.
try:
    result = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero!")
מתודה אבסטרקטית (AbstractMethod)
פונקציה שהיא "חוזה" ריק. היא מכריחה מחלקות-בנות ליישם אותה (למשל כל סוג של Repository יהיה חייב להחזיק מתודת save).
from abc import ABC, abstractmethod

class BaseRepo(ABC):
    @abstractmethod
    def save(self, data):
        pass # לא עושים כלום כאן
פריקת מילון (**kwargs)
לוקח מילון ושופך את כל תוכנו כפרמטרים נפרדים לתוך קריאה לפונקציה.
my_dict = {"name": "Natan", "age": 20}
# שקול ל: create_user(name="Natan", age=20)
create_user(**my_dict)
קו תחתון פנוי (_)
הסכמה בפייתון שאומרת: "המשתנה הזה נדרש תחבירית, אבל אני מתכוון להתעלם ממנו ולא להשתמש בו".
# אנחנו עושים לולאה של 5 פעמים, אבל ה'אינדקס' לא מעניין אותנו
for _ in range(5):
    print("Hello!")
